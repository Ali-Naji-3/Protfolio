# ALI NAJI — Cinematic Personal Brand Platform
## Production Package v2.0

An interactive experience — a scroll-driven film set **inside a living backend system** — for **Ali Naji, early-career backend Software Engineer.**

> **The one sentence the visitor must remember:**
> *"Engineering is often about preventing invisible failures rather than building visible features."*

**Status:** Pre-production. Creative Constitution ratified in the strategy interview. No implementation code yet, by design. v2 supersedes v1 in full.

---

## The Creative Constitution (ratified)

| | |
|---|---|
| **Positioning** | Early-career backend engineer, honest, instincts beyond his years — no staff/executive cosplay |
| **Thesis** | Preventing invisible failures rather than building visible features |
| **World** | The inside of a living backend system — architecture as landscape, data as atmosphere, motion as explanation. No place, no lobby |
| **Hook** | The race condition — emotion in 3s, technical depth optional. Dual legibility |
| **Motif** | One philosophy, many self-built proofs. Private projects are credentials, not exhibits |
| **Human** | The real Ali (identity-validated), placed inside the system as its author |
| **Audio** | In, optional, off by default. Precision sound, no score. Muted experience complete |
| **Production** | Small budget, weeks of time. AI renders the world; real photography for identity. No film shoot |

### Core Laws (every document answers to these)
1. **Beauty serves clarity** — every visual decision must explain an engineering idea; never decoration.
2. **The medium is the argument** — the site's own engineering is the proof.
3. **The story is true; the visuals look like the real Ali.**
4. **Confidentiality is absolute** — no proprietary client material, ever.
5. **Dual legibility** — emotion for the recruiter, precision for the engineer.
6. **80% calm / 20% epic.**
7. **Silence is a material.**

## Hierarchy of authority

`01` overrides all. `07` has veto over any human/confidential asset. `02` defines *what happens*; `03`/`04` define *how it moves and looks*; `05`/`06`/`08` serve them; `09` is the source of truth for all engineering content. Conflicts escalate to the Creative Producer.

## Before build begins (critical path)

1. **Capture the ground-truth identity package** (`05` §3.1 / `07`) — everything human depends on it.
2. **Finalize Proof #2 & #3 content** (`09` §4–5) — Chapter 04 and the demos are blocked on it.
3. **Build DEMO-01** (the race condition) — the hook and the proof-of-format.
4. Delivery manifest satisfied (`05` §10).

**Definition of done:** a studio's engineering team could build the site from these documents without a single creative question — and without one proprietary byte entering the repository.
